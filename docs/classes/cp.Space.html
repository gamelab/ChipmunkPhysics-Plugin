Properties

collisionBias

Number public

Defined in src/chipmunk-docs/space.js:61

Determines how fast overlapping shapes are pushed apart. Expressed as a fraction of the error remaining after each second. Defaults to pow(1.0 - 0.1, 60.0) meaning that Chipmunk fixes 10% of overlap each frame at 60Hz.

Default: 1

collisionPersistence

Number public

Defined in src/chipmunk-docs/space.js:72

Number of frames that contact information should persist. Defaults to 3. There is probably never a reason to change this value.

Default: 3

collisionSlop

Number public

Defined in src/chipmunk-docs/space.js:49

Amount of encouraged penetration between colliding shapes.. Used to reduce oscillating contacts and keep the collision cache warm. Defaults to 0.1. If you have poor simulation quality, increase this number as much as possible without allowing visible amounts of overlap.

Default: 0.1

damping

Number public

Defined in src/chipmunk-docs/space.js:27

Damping rate expressed as the fraction of velocity bodies retain each second. A value of 0.9 would mean that each body's velocity will drop 10% per second.

Default: 1

gravity

Object public

Defined in src/chipmunk-docs/space.js:18

Gravity to pass to rigid bodies when integrating velocity.

Default: {x:0,y:0}

idleSpeedThreshold

Number public

Defined in src/chipmunk-docs/space.js:82

Speed threshold for a body to be considered idle. The default value of 0 means to let the space guess a good threshold based on gravity.

Default: 0

iterations

Number public

Defined in src/chipmunk-docs/space.js:38

The number of iterations that the space property should update each frame. The higher this is the more accurate the physics will be, but the more processor intensive it will be.

Default: 10
